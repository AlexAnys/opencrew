# v2-lite — 精简版架构（探索方向）

> **状态**：实验性设计，尚未完全部署验证。仅供参考和讨论。

## v1 → v2 的核心变化

| 维度 | v1 (当前稳定版) | v2 (探索方向) |
|------|----------------|---------------|
| Agent 数量 | 7 | 5（KO→cron, Ops→CoS 兼任） |
| 每 Agent 文件 | 8 | 4-5 |
| Shared 协议 | 9 文件 | 3 文件 |
| 治理方式 | Ops 独立 Agent | CoS 兼任周期性治理 |
| 知识提取 | KO 独立 Agent | cron 定时子任务 |
| 硬约束执行 | 文档 + 配置 | 更多下沉到配置 |
| CoS 派单 | 必须经 CTO | 灵活：简单任务可直派 Builder |
| 自我迭代 | 6 字段模板 + Ops 审批 | 一行日记 + 周检 |
| Signal 评分 | 0-3 梯度 | 二值"值得沉淀" |

## 为什么探索 v2？

v1 的 9 个 shared 文件中，很多约束靠"Agent 自觉遵守"——实际运行中发现，**配置层的硬约束比文档层的软约束可靠得多**。

v2 的核心思路：
- 能写进 config 的约束就不写进文档
- 能用 cron 自动化的就不依赖 Agent 主动执行
- 减少 Agent 间的协调成本

## v2 的 3 个 Shared 文件

1. **RULES.md** — 核心规则（31 行，合并了 SYSTEM_RULES + OPS_REVIEW 的关键条目）
2. **A2A.md** — 协作协议（38 行，精简版两步触发）
3. **TEMPLATES.md** — 所有模板合一（70 行，Closeout + Checkpoint + Task Packet）

## v2 的 5 个 Agent

| Agent | 变化 |
|-------|------|
| CoS | 新增治理职责（周检 + 月审） |
| CTO | 不变 |
| Builder | 新增"When Blocked Protocol" |
| CIO | 不变（仍为可选） |
| Research | 不变（仍为 spawn-only） |

## 待验证的问题

## 使用方式

v2 目录结构独立，可以直接替换 v1 的对应文件进行实验。建议先备份 v1 配置再尝试。
